Repo de hoy: https://github.com/compiladores/esqueleto-jsonlang#

Brevísimo resumen de semánticas formales#

  • Semántica axiomática (Lógica de Hoare): expresa la semántica por medio de pre y post condiciones
  • Semántica operacional: como la axiomática pero con más conceptos, que modelan conceptos como la RAM, programas posibles, etc. como conjuntos.
  • Semántica denotacional: Interpreta el programa como una función matemática que involucra la RAM, contextos, etc.

Triplas de Hoare#

{P}C{Q}\{P\}C\{Q\}{P}C{Q}

Al ejecutar CCC, si el estado del programa es PPP, pasa a ser QQQ

Ejemplo#

Asignación en C

{S} int x = 4; {S∧x=4}\{S\} \texttt{ int x = 4; } \{S \land x=4\}{S} int x = 4; {S∧x=4}
{S} int x = N; {S∧x=N}\{S\} \texttt{ int x = $N$; } \{S \land x=N\}{S} int x = N; {S∧x=N}

Secuencia:

{A}S1{B},{B}S2{C}⇒{A}S1;S2{C}\{A\} \texttt{$S_1$} \{B\}, \{B\} \texttt{$S_2$} \{C\} \Rightarrow \{A\} \texttt{$S_1$;$S_2$} \{C\}{A}S1​{B},{B}S2​{C}⇒{A}S1​;S2​{C}

Semántica Denotacional#

Cada construcción sintáctica del programa se convierte en una representación matemática rigurosa de lo que hace.

Ejemplo#

En C, todo el código que puede ejecutarse es un statement, así, podemos tener:

  • if statement
  • while statement
  • block statement
  • assign statement
  • expression statement

Una función semántica va a ser será una F:Statement→(RAM×disk→RAM×disk)F: Statement \to (RAM \times disk \to RAM \times disk)F:Statement→(RAM×disk→RAM×disk)

Jsonlang: un lenguaje de POJOs#

  • JJJ: valores javascript válidos
  • expr⊂Jexpr\subset Jexpr⊂J: expresiones del lenguaje
  • C:string→JC:string\to JC:string→J: diccionario contexto
  • S:N→JS:\N\to JS:N→J: lista
  • Si⊂SS_i\subset SSi​⊂S inputs, So⊂SS_o\subset SSo​⊂S outputs

Sea la función semántica:

F:expr×D×Si×So→J×D×Si×SoF:expr \times D \times S_i \times S_o\to J \times D \times S_i \times S_oF:expr×D×Si​×So​→J×D×Si​×So​

Esta función semántica vamos a llamarla eval. Ejemplos en el código.
